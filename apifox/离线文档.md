# 聊天补全

## POST 创建

POST /v1/chat/completions

若调用模型列表不存在的名单，则直接拒绝
豆包lite，mini模型不支持温度参数（会默认忽略）
Qwen3.6 默认开启思考模式 所以说回复会慢些（没时间搞思考开关 所以说无法跳过 TNT）

> Body 请求参数

```json
{
  "model": "gpt-4o-mini",
  "messages": [
    { "role": "system", "content": "You are a helpful assistant." },
    { "role": "user", "content": "Hello!" }
  ],
  "temperature": 0.7,
  "stream": false
}
```

### 请求参数

|名称|位置|类型|必选|中文名|说明|
|---|---|---|---|---|---|
|Accept|header|string| 是 ||none|
|body|body|object| 是 ||none|
|» model|body|string| 是 ||模型型号|
|» messages|body|[object]| 是 | 消息列表|当前对话中的所有消息列表。根据 您使用的模型，支持不同的消息类型（模式），例如文本、 图像和音频。|
|»» role|body|string| 是 | 角色|none|
|»» content|body|string| 是 | 内容|none|
|» temperature|body|number| 是 | 采样温度|采样温度应设置为 0 到 2 之间的值。较高的值（例如 0.8）会使输出结果更随机，而较低的值（例如 0.2）会使输出结果更集中、更确定。|
|» stream|body|boolean| 是 | 是否流式返回|如果设置为 true，模型响应数据将使用服务器发送事件以流式传输的方式在生成时发送给客户端。|

> 返回示例

> 模型根据所提供的输入返回的聊天完成响应

### 返回结果

|状态码|状态码含义|说明|数据模型|
|---|---|---|---|
|200|[OK](https://tools.ietf.org/html/rfc7231#section-6.3.1)|模型根据所提供的输入返回的聊天完成响应|Inline|

### 返回数据结构

状态码 **200**

|名称|类型|必选|约束|中文名|说明|
|---|---|---|---|---|---|
|» id|string|true|none||用于标识聊天记录完成的唯一标识符|
|» object|string|true|none|类型标识符|非流式："chat.completion"，流式："chat.completion.chunk"|
|» created|integer|true|none|响应创建时间|聊天补全响应生成的 Unix 时间戳（秒级）|
|» model|string|true|none|模型|用于聊天补全的模型|
|» choices|[object]|true|none|聊天完成选项列表|none|
|»» index|integer|false|none|该选项在选项列表中的索引|none|
|»» message|object|false|none|聊天补全消息|none|
|»»» role|string|true|none|角色|none|
|»»» content|string|true|none|内容|none|
|»» finish_reason|string|false|none|模型停止生成令牌的原因|可能是stop模型遇到了自然停止点或预设的停止序列， length也可能是请求中指定的最大令牌数已达到， content_filter或者由于内容过滤器中的标志而省略了内容， tool_calls或者模型调用了工具，或者function_call（已弃用）模型调用了函数。|
